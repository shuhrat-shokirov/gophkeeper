
# 🧰 Go Service Template

Этот проект — шаблон для написания модульных REST API сервисов на языке Go. Он построен по принципам чистой архитектуры и предполагает чёткое разделение ответственности по слоям.

---

## 📐 Архитектура проекта

Структура проекта отражает принципы масштабируемости и поддержки:

```
gophkeeper/
├── .github/
│ └── workflows/ # GitHub Actions
│   ├── lint.yml # Линтер
│   └── unit-tests.yml # Юнит-тесты
│
├── cmd/ # Точка входа в приложение
│ ├── main.go # Основной файл приложения
│
├── configs/ # Конфигурация приложения
│ ├── config.json # Конфигурация сервиса
│
├── internal/ # Внутренние модули (изолированы от внешнего мира)
│ ├── gateways/ # Внешние клиенты (например, API других сервисов)
│ ├── repositories/ # Доступ к хранилищам данных
│ ├── services/ # Бизнес-логика
│ └── rest/ # HTTP слой
│   ├── handlers/ # Обработчики эндпоинтов
│   ├── middleware/ # Middleware-функции
│   └── routes/ # Маршруты API
│
├── Makefile # Утилиты запуска
├── run_me.sh # Скрипт инициализации проекта
├── .golangci.yml # Настройка линтера
├── go.mod / go.sum # Управление зависимостями
└── README.md
```

Такой подход позволяет:
- 🚀 легко масштабировать проект,
- 🧪 удобно писать и запускать тесты,
- 🧱 независимо развивать слои,
- 🔒 минимизировать связность между модулями.

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-org/your-service.git
cd your-service
```

### 2. Инициализация проекта
Дать права на run_me.sh и запустить
```
chmod +x run_me.sh
./run_me.sh
```

Скрипт автоматически:
заменит все gophkeeper на имя вашего репозитория;
удалит сам себя;
выведет список TODO, которые нужно учесть.

### ⚙️ Конфигурация

Файл configs/config.json содержит параметры окружения:
```
{
  "server_port": "8080",
  "log_level": "debug"
}
```

### ✅ GitHub Actions

CI настроен через .github/workflows/:
```
lint.yml — прогоняет golangci-lint
unit-tests.yml — запускает все юнит-тесты.
```
